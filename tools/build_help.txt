sloppy-ethos build.py command reference

USAGE
  python tools/build.py [options]

PRIMARY ACTIONS
  --dist                 Build install ZIP(s) in dist/.
  --deploy               Deploy scripts/{ProjectName} and optional manifest-declared radio files to the simulator path.
  --clean                Remove deployed scripts/{ProjectName}, manifest-declared radio files, and clear dist/ (or --out-dir) artifacts.

COMMON OPTIONS
  --project, -p NAME     Project folder under scripts/. Repeat for multi-project dist bundles.
  --config, -c PATH      Deploy config path. Default: tools/deploy.config.json.
  --sim-radio MODEL      Resolve simulator path from ETHOS_SIM_PATHS entry for MODEL.
  --out-dir PATH         ZIP output directory. Default: dist/.
  --version VERSION      Override script VERSION for single-project --dist only.
  --no-zip               Skip ZIP creation even when --dist is provided.
  --help                 Print this command reference and exit.

ZIP NAMING
  Single-project --dist:
    dist/{ProjectName}-{version}.zip
    where version comes from scripts/{ProjectName}/VERSION (or --version override)

  Multi-project --dist (repeat --project):
    dist/sloppy-ethos_scripts.zip
    multi-script bundle ZIP name is intentionally unversioned

PROJECT BUILD MANIFEST
  Projects may include a local build.json file with a radioFiles array:
    {
      "radioFiles": [
        { "source": "extras/widget-icon.png", "destination": "bitmaps/widgets/widget-icon.png" }
      ]
    }
  source paths are relative to the project directory under scripts/.
  destination paths are relative to the radio root and must stay outside scripts/.
  Declared radio files are packaged into ZIPs and are deployed/cleaned with the project.

  Projects with local map folders may instead declare generic mapAssets:
    {
      "mapAssets": [
        {
          "source": "maps",
          "bitmapDestination": "bitmaps/GPS",
          "metadataDestination": "documents/user"
        }
      ]
    }
  mapAssets source directories are relative to the project directory.
  Missing mapAssets source directories are allowed so public checkouts can build without private maps.
  Discovered BMP/PNG files install to bitmapDestination; discovered JSON files install to metadataDestination.
  Other generator outputs, including metadata text files, are ignored by mapAssets.

SIMULATOR PATH RESOLUTION
  With --sim-radio:
    1) Match ETHOS_SIM_PATHS entry by "radio"

  Without --sim-radio:
    1) Use ETHOS_SIM_PATHS entry with "default": true
    2) Exactly one default entry is allowed

CONFIG EXAMPLE
  {
    "ETHOS_SIM_PATHS": [
      { "radio": "X20RS", "path": "C:\\Users\\your-user\\AppData\\Roaming\\Ethos Suite\\.simulator\\1.6.4\\persist\\X20RS", "default": true },
      { "radio": "X20S", "path": "C:\\Users\\your-user\\AppData\\Roaming\\Ethos Suite\\.simulator\\1.6.4\\persist\\X20S" }
    ]
  }

WORKFLOW EXAMPLES
  Build install ZIP:
    python tools/build.py --project SensorList --dist

  Build BoundryMap ZIP, including any local maps under scripts/BoundryMap/maps/:
    python tools/build.py --project BoundryMap --dist

  Build multi-script install ZIP bundle:
    python tools/build.py --project SensorList --project ethos_events --dist

  Deploy to default simulator path:
    python tools/build.py --project SensorList --deploy

  Deploy to model-specific simulator path:
    python tools/build.py --project SensorList --deploy --sim-radio X20RS

  Clean deployed scripts from model-specific simulator path:
    python tools/build.py --project SensorList --clean --sim-radio X20RS

  Run clean then deploy in one command:
    python tools/build.py --project SensorList --clean --deploy
