# Security Policy

## Reporting a Vulnerability

Please do not open public GitHub issues for suspected security vulnerabilities.

Report security concerns privately to the maintainers and include:

- summary of the issue and impact
- affected files/components
- steps to reproduce
- suggested mitigation (if known)

If direct maintainer contact is unavailable, open a minimal issue asking for a private reporting channel without disclosing exploit details.

## Response Expectations

- Initial acknowledgment target: within 7 days.
- Triage and next-step status target: within 14 days.
- Fix/release timing depends on severity and reproduction confidence.

## Scope

This policy covers:

- repository source and tooling (`src/`, `tools/`, `tests/`)
- build/deploy workflow documentation
- packaged widget artifacts generated from this repository
